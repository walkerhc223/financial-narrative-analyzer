Financial Narrative Analyzer
Logic Flow v1

Purpose:
Define the end-to-end sequence of how the system processes raw financial inputs into a structured financial narrative.

1. Receive Input
- Accept structured raw financial inputs
- Accept optional management notes and materiality thresholds

2. Validate Input
- Check that required fields are present
- Check that required values are numeric
- Check for invalid or undefined inputs
- Store validation flags if needed

3. Compute Metrics
- Calculate absolute changes
- Calculate percentage changes
- Calculate margins
- Calculate efficiency ratios
- Calculate leverage/conversion signals

4. Apply Edge Case Logic
- Handle divide-by-zero cases
- Flag undefined percentage changes
- Preserve negative values where relevant
- Carry forward any calculation warnings

5. Interpret Signals
- Apply directional rules
- Apply relative growth rules
- Apply materiality rules
- Apply signal ranking rules
- Apply causal pattern rules
- Use management notes only as supporting context

6. Select Main Narrative Story
- Determine highest-priority material signal
- Determine supporting signals
- Identify whether the story is primarily growth, cost pressure, margin pressure, efficiency improvement, or deterioration

7. Generate Narrative Components
- Create executive summary points
- Create revenue commentary
- Create gross profit commentary
- Create operating expense commentary
- Create operating income conclusion
- Create optional key watchout if supported

8. Assemble Final Output
- Combine narrative sections into defined output structure
- Ensure ordering reflects signal priority
- Ensure wording stays concise, factual, and management-oriented

9. Return Final Result
- Return structured narrative
- Return computed metrics if needed for transparency/debugging
- Return validation or calculation flags if relevant
