Financial Narrative Analyzer
Module Outline v1

Purpose:
Define the major system modules and their responsibilities for the Financial Narrative Analyzer.

--------------------------------------------------
1. input_handler
--------------------------------------------------

Responsibility:
Receive and organize raw structured financial input data.

Key duties:
- Accept required financial fields
- Accept optional context fields
- Standardize input format
- Pass input data to validation module

Possible functions:
- load_input_data()
- normalize_input_data()

--------------------------------------------------
2. validator
--------------------------------------------------

Responsibility:
Check that the input data is complete, usable, and structurally valid before computation begins.

Key duties:
- Verify required fields are present
- Verify numeric fields are numeric
- Apply default values for optional thresholds if missing
- Flag missing or invalid inputs
- Prevent invalid data from entering downstream modules

Possible functions:
- validate_required_fields()
- validate_numeric_fields()
- apply_default_thresholds()
- validate_inputs()

--------------------------------------------------
3. computation_engine
--------------------------------------------------

Responsibility:
Transform raw financial inputs into computed financial metrics.

Key duties:
- Calculate absolute changes
- Calculate percentage changes
- Calculate margins
- Calculate efficiency ratios
- Calculate leverage/comparison signals
- Handle divide-by-zero safely
- Return structured computed metrics

Possible functions:
- compute_absolute_changes()
- compute_percentage_changes()
- compute_margins()
- compute_efficiency_ratios()
- compute_growth_gap_signals()
- safe_divide()
- compute_metrics()

--------------------------------------------------
4. interpretation_engine
--------------------------------------------------

Responsibility:
Convert computed metrics into structured financial signals and causal patterns.

Key duties:
- Apply directional rules
- Apply relative growth rules
- Apply materiality rules
- Apply signal ranking rules
- Apply causal pattern rules
- Incorporate management notes only when supported
- Select primary headline signal
- Organize favorable, unfavorable, material, and supporting signals

Possible functions:
- apply_directional_rules()
- apply_relative_growth_rules()
- apply_materiality_rules()
- apply_signal_ranking_rules()
- apply_causal_pattern_rules()
- determine_primary_headline_signal()
- interpret_metrics()

--------------------------------------------------
5. narrative_generator
--------------------------------------------------

Responsibility:
Translate interpreted financial signals into management-ready narrative sections.

Key duties:
- Generate executive summary
- Generate revenue commentary
- Generate gross profit commentary
- Generate operating expense commentary
- Generate operating income conclusion
- Generate key watchout if applicable
- Keep tone concise, factual, and management-oriented

Possible functions:
- generate_executive_summary()
- generate_revenue_commentary()
- generate_gross_profit_commentary()
- generate_operating_expense_commentary()
- generate_operating_income_conclusion()
- generate_key_watchout()
- generate_narrative_sections()

--------------------------------------------------
6. report_assembler
--------------------------------------------------

Responsibility:
Assemble individual narrative sections into the final structured output.

Key duties:
- Order sections correctly
- Omit optional sections if not needed
- Format final report consistently
- Return final narrative output

Possible functions:
- assemble_final_output()
- format_report()

--------------------------------------------------
7. output_handler
--------------------------------------------------

Responsibility:
Return or display the final system output and optional supporting information.

Key duties:
- Return final structured narrative
- Optionally return computed metrics
- Optionally return interpretation results
- Optionally return validation/calculation flags for debugging

Possible functions:
- build_output_payload()
- return_final_result()

--------------------------------------------------
8. orchestrator
--------------------------------------------------

Responsibility:
Coordinate the full end-to-end workflow across all modules.

Key duties:
- Call validation module
- Call computation engine
- Call interpretation engine
- Call narrative generator
- Call report assembler
- Return final result
- Manage overall process order

Possible functions:
- financial_narrative_analyzer()

--------------------------------------------------
9. Suggested Dependency Flow
--------------------------------------------------

input_handler
    ->
validator
    ->
computation_engine
    ->
interpretation_engine
    ->
narrative_generator
    ->
report_assembler
    ->
output_handler

The orchestrator coordinates the full sequence.

--------------------------------------------------
10. Version 1 Design Notes
--------------------------------------------------

- Keep modules loosely coupled
- Keep computation deterministic
- Keep interpretation rule-based before narrative generation
- Use the LLM primarily for wording, not for calculation
- Design modules so future versions can add balance sheet, cash flow, and multi-period analysis without redesigning the full system
